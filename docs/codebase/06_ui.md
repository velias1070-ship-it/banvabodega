## Sistema de diseño

**No usa ninguna component library** (no shadcn, no MUI, no Radix). Todo es CSS custom.

### Theme dark obligatorio
Variables CSS en `:root` de `src/app/globals.css` (ver `.claude/rules/ui-ux.md`):
- Fondos: `--bg`, `--bg2`, `--bg3`, `--bg4` (`#0a0e17` → `#243049`).
- Texto: `--txt`, `--txt2`, `--txt3`.
- Colores semánticos: `--green`, `--red`, `--blue`, `--amber`, `--cyan` con variantes `Bg` (10% opacity) y `Bd` (25% opacity).

### Fuentes
- **Outfit** (sans-serif) — UI.
- **JetBrains Mono** — datos numéricos, SKUs, códigos. Clase `.mono`.

### Clases CSS principales
`.card`, `.topbar`, `.tabs/.tab`, `.kpi/.kpi-grid`, `.tbl`, `.scan-btn` (con variantes `.green/.blue/.red`), `.form-input/.form-label`, `.qty-row/.qty-btn`, `.app-admin`, `.admin-sidebar`, `.sidebar-btn`, `.admin-layout`.

### Iconos
Emojis nativos. No icon library.

### Mobile-first (operador)
- `.app` con `max-width: 480px`, `min-height: 100dvh`.
- `userScalable: false` en viewport meta — comportamiento PWA.
- Manifest en `public/manifest.json` (instalable como app).
- `padding-bottom: env(safe-area-inset-bottom)`.

### Admin-first
- `.app-admin` full width.
- Sidebar con accordion de grupos (un grupo abierto a la vez).
- Topbar con fecha y botón cerrar sesión.

## Patrones de estado

**No hay estado global** (no Zustand, no Redux, no Context para datos de dominio).

- **Estado local** con `useState` en cada componente.
- **`src/lib/store.ts`** (~3 100 LOC) actúa como **bridge en memoria + cache de la DB**. Expone funciones imperativas (`getStore`, `saveStore`, `refreshStore`, `recordMovement`, `findProduct`, `crearRecepcion`, etc.) y mantiene un objeto en memoria que se sincroniza con Supabase.
- **Refresh pattern** explícito: `const [,setTick] = useState(0); const r = () => setTick(t=>t+1);`. Los hijos llaman `r()` para forzar re-render del padre.
- **Polling** cada 10s: `setInterval(async () => { await refreshStore(); r(); }, 10_000)` — reemplaza React Query.
- **`sessionStorage`** para persistencia entre tabs/recargas (auth, last tab, SII creds).

> 14 archivos importan `@/lib/store`. Es el "estado global" de facto. Discutir migración a un store reactivo (Zustand) o a queries/mutations sería un cambio grande.

## Hallazgos UI

1. **`src/app/admin/page.tsx` es un megacomponente de 12 201 líneas**. Contiene `Dashboard`, `AdminRecepciones`, `AdminUltimaMilla`, `AdminEnviosFull`, `Operaciones`, `Inventario`, `Movimientos`, `Productos`, `AdminTimeline`, `Configuracion`, `MobileMenu` como funciones internas. Refactor candidato evidente.
2. **Inline styles extensivos**: `style={{}}` con camelCase es la norma. CSS classes solo para patrones repetidos (regla en `.claude/rules/ui-ux.md`).
3. **`useAuth` duplicado** entre `/admin` y `/conciliacion` con el mismo `ADMIN_PIN = "1234"` hardcodeado.
4. Existe `lib/admin-users.ts` con `loginAdminUser`, `canAccessTab`, etc. — sistema de usuarios v61 — pero el PIN antiguo sigue conviviendo.
5. **No hay design tokens en JS**: los colores viven solo en CSS variables, sin export TS. Cualquier componente que necesite colores semánticos los toma con `var(--green)` desde inline style.
